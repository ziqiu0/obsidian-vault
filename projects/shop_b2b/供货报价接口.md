# 供货报价接口 (SupplyOffer)

基础路径: `/api/offers`

## 供应商接口 (需要 supplier 角色)

### POST / - 创建报价

**需要认证:** 是 (supplier)

**请求体:**
```json
{
  "demandId": "采购需求ID",
  "price": 单价,
  "quantity": 可供应数量,
  "deliveryTime": 交货天数,
  "remark": "备注说明"
}
```

**响应:** 创建成功的报价

### GET /my - 获取我的报价列表

**需要认证:** 是 (supplier)

**响应:** 该供应商所有报价

### POST /:id/withdraw - 撤回报价

→ `withdrawn`

## 管理接口 (需要 admin/procurement 角色)

### GET /pending - 获取待审核报价列表

**需要认证:** 是 (admin/procurement)

### POST /:id/approve - 通过报价

**需要认证:** 是 (admin/procurement)

自动创建 [[供货订单模型 (SupplyOrder)]]，并关闭原采购需求。

### POST /:id/reject - 拒绝报价

**需要认证:** 是 (admin/procurement)

报价 → `rejected`

## 公开查询

### GET /demand/:demandId - 获取需求的报价列表

**需要认证:** 是

- 管理员: 查看所有报价
- 供应商: 只能看到自己的报价
