# 采购需求接口 (ProductDemand)

基础路径: `/api/demands`

## 公开接口 (所有已登录用户可访问)

### GET / - 获取采购需求列表

**需要认证:** 是

**查询参数:**
- `page`: 页码
- `limit`: 每页数量
- `status`: 状态筛选（可选）
- `keyword`: 关键词搜索（可选）

**响应:**
```json
{
  "code": 200,
  "data": {
    "demands": [...],
    "total": 总数,
    "page": 1,
    "totalPages": ...
  }
}
```

### GET /:id - 获取需求详情

**需要认证:** 是

**响应:**
```json
{
  "code": 200,
  "data": {
    "demand": { ... },
    "offers": [...]  // 包含已提交的报价（管理员可见全部，供应商只看自己的）
  }
}
```

## 管理接口 (admin/procurement 权限)

### POST / - 创建采购需求

**需要认证:** 是 (admin/procurement)

**请求体:**
```json
{
  "title": "需求标题",
  "description": "详细描述",
  "productName": "商品名称",
  "quantity": 1000,
  "targetPrice": 50,
  "category": "分类",
  "requiredDeliveryDate": "2026-06-01",
  "images": [...]
}
```

**响应:** 创建成功的需求

### PUT /:id - 更新采购需求

**需要认证:** 是 (admin/procurement)

**请求体:** 待更新字段

### POST /:id/submit - 提交审核

将 `draft` → `pending_review`

### POST /:id/publish - 发布需求

审核通过后发布 → `published`

### POST /:id/close - 关闭需求

→ `closed`

### GET /my - 获取我创建的需求

**需要认证:** 是
