# 三级记忆架构

## 什么是三级记忆架构

为了解决 Context Window 限制而设计的**分级记忆系统**，将信息按热度分层管理：

| 层级 | 存储位置 | 内容 | 特点 |
|------|---------|------|------|
| 🔵 热层 | Working Memory | 最近 N 轮对话 | 完整保留，不压缩 |
| 🟢 温层 | `memory/daily/` | 每日日志 | 早期对话压缩摘要 |
| 🟣 冷层 | `memory/semantic/` | 向量索引 | 全量历史，按需检索 |

## 为什么需要它

当以下内容加起来超过 Context Window 上限时：
- 系统 Prompt / Soul
- Skill 列表
- 对话历史
- 工具调用记录
- 记忆文件

模型会**溢出失忆**或报错。

## 在 OpenClaw 中的实现

```
workspace/
├── memory/                    ← 三级记忆存储
│   ├── daily/               ← 🟢 温层：每日笔记
│   ├── summaries/           ← 🟢 温层：摘要
│   └── semantic/            ← 🟣 冷层：向量索引
├── MEMORY.md                ← 长期记忆（核心索引）
└── obsidian-vault/          ← Obsidian 笔记同步
```

## 工程手段

| 手段 | 说明 |
|------|------|
| Skill 白名单 | 只加载必要的 Skill，减少系统 Prompt |
| 历史窗口截断 | 只保留最近 N 轮 |
| 结果压缩 | 工具返回前先提炼摘要 |
| 分模型路由 | 简单任务用小模型，省 Context |
